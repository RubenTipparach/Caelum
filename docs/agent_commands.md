# Agent Command Reference

Commands are typed in the chat window (press E near an agent). Commands are processed locally — no LLM call needed. Everything else goes to the AI for conversation.

## Movement Commands

| Command | Variations | What it does |
|---------|-----------|-------------|
| `follow me` | `follow`, `come with me`, `come here`, `come along` | Agent follows the player continuously. Runs at 8m/s if >30m away. |
| `go <dir> <N>` | `walk <dir> <N>`, `move <dir> <N>` | Walk N cells in a direction. |
| `stay` | `stop`, `wait here`, `hold` | Stop everything, go idle. |

### Directions
`north`, `south`, `east`, `west`, `ne`, `nw`, `se`, `sw`, `forward`, `ahead`
Short forms: `n`, `s`, `e`, `w`

## Stop Commands

Any of these will halt all current activity (walking, building, following, scripts):

- `stop`, `stay`, `wait here`, `hold`
- `stop building`, `stop following`, `stop working`, `stop walking`, `stop moving`
- `stop construction`, `cancel`, `abort`, `nevermind`
- `knock it off`, `that's enough`

## Build Commands (Simple — Instant)

| Command | Example | What it does |
|---------|---------|-------------|
| `build wall <dir> <N>` | `build wall north 5` | Build N-block wall in direction |
| `build stairs <dir> <N>` | `build stairs east 3` | Build N-step staircase |
| `build wall <dir> <N> <block>` | `build wall north 5 dirt` | Build with specific block type |

### Block Types
`stone` (default), `dirt`, `grass`, `sand`, `ice`, `torch`

## Build Commands (Complex — LLM Generated)

If you say "build" + something that isn't a simple wall/stairs, the AI generates a build script:

- `build a house`
- `build a tower`
- `build a bridge going east`
- `construct a wall around me`
- `make me a shelter`
- `create a staircase to the top`
- `dig a hole`, `mine down 5 blocks`
- `flatten this area`, `demolish that wall`

The agent will respond with a short acknowledgment, generate a JSON script with placement/break steps, save it to `cache/agents/<name>/orders/`, and execute it.

## Terrain Commands

| Command | What it does |
|---------|-------------|
| `clear area <N>` | Break all blocks above ground in N-cell radius |
| `clear ahead <N>` | Clear blocks ahead for N cells (cut a path) |
| `clear <dir> <N>` | Clear in a specific direction |

## System Commands

| Command | What it does |
|---------|-------------|
| `/scan` | Run sensor scan, print results to console |
| `/script <name>` | Load and run `cache/scripts/<name>.json` |

## Conversation

Anything not matching the above patterns goes to the AI as normal chat. The AI receives:
- Your message
- Current mood/emotions
- Terrain sensor data (nearby blocks, elevation, walkability)
- Persistent memory (from `cache/agents/<name>/memory.md`)
- Recent journal events

### Greetings (triggers wave animation)
`hi`, `hey`, `hello`, `yo`, `sup`, `what's up`, `whats up`, `how are you`, `howdy`, `greetings`, `hola`

### Memory
The agent can save important things you tell it using `[REMEMBER]...[/REMEMBER]` tags in its response. These persist in `memory.md` and are loaded on every startup.

## Scripting API

Scripts are JSON files that tell an agent to perform a sequence of actions. They can be:
- **Generated automatically** from simple orders (`build wall north 5`)
- **Generated by the LLM** from complex requests (`build a house`)
- **Hand-written** and loaded with `/script <name>`

### Coordinate System

The hex planet uses three coordinates:

- **q, r** — hex grid column and row. Like Minecraft's x,z but for hexagons. Each hex cell is ~1.7m wide.
- **layer** — vertical position (like Minecraft's y). Each layer = 1 meter. Layer 0 = bedrock, higher = above ground.

```
Example: agent is at q=57, r=-73, ground_layer=5192

        layer 5194  ← air
        layer 5193  ← 1m above ground (place blocks here for walls)
        layer 5192  ← ground surface (grass/dirt/stone)
        layer 5191  ← underground
```

### Script JSON Format

Scripts are saved to `cache/agents/<name>/orders/` or `cache/scripts/`.

```json
{
  "name": "my_wall",
  "description": "Build a 3-block stone wall going east",
  "steps": [
    {"action": "move_to", "q": 58, "r": -73},
    {"action": "place", "q": 58, "r": -73, "layer": 5193, "block": "stone"},
    {"action": "move_to", "q": 59, "r": -73},
    {"action": "place", "q": 59, "r": -73, "layer": 5193, "block": "stone"},
    {"action": "move_to", "q": 60, "r": -73},
    {"action": "place", "q": 60, "r": -73, "layer": 5193, "block": "stone"},
    {"action": "say", "text": "Wall's done!"}
  ]
}
```

### Blueprints (Relative Coordinates)

For portable structures that work anywhere on the planet, use relative actions with an anchor point.
The anchor is automatically set to the agent's position when the script starts, or explicitly with `set_anchor`.

```json
{
  "name": "hex_hut",
  "description": "A small hex hut - works anywhere",
  "steps": [
    {"action": "set_anchor"},
    {"action": "place_rel", "dq": 0, "dr": 0, "dlayer": 1, "block": "stone"},
    {"action": "place_rel", "dq": 1, "dr": 0, "dlayer": 1, "block": "stone"},
    {"action": "place_rel", "dq": 0, "dr": 1, "dlayer": 1, "block": "stone"},
    {"action": "place_rel", "dq": 1, "dr": 1, "dlayer": 1, "block": "stone"},
    {"action": "place_rel", "dq": 0, "dr": 0, "dlayer": 2, "block": "stone"},
    {"action": "place_rel", "dq": 1, "dr": 0, "dlayer": 2, "block": "stone"},
    {"action": "place_rel", "dq": 0, "dr": 1, "dlayer": 2, "block": "stone"},
    {"action": "place_rel", "dq": 1, "dr": 1, "dlayer": 2, "block": "stone"},
    {"action": "say", "text": "Hut's done!"}
  ]
}
```

`dq`, `dr`, `dlayer` are offsets from the anchor point. `dlayer: 1` = one block above ground, `dlayer: 2` = two above, etc.

For mega-structures, the agent walks to each position via `move_rel` before placing, ensuring chunks load as needed:
```json
{"action": "move_rel", "dq": 50, "dr": 0},
{"action": "place_rel", "dq": 50, "dr": 0, "dlayer": 1, "block": "stone"}
```

### Available Script Actions

| Action | Parameters | Description |
|--------|-----------|-------------|
| `move_to` | `q`, `r` | Pathfind and walk to a hex cell |
| `place` | `q`, `r`, `layer`, `block` | Place a block at the position |
| `break` | `q`, `r`, `layer` | Break/remove a block |
| `say` | `text` | Display a message in chat |
| `wait` | `seconds` | Pause for N seconds |
| `jump` | — | Jump in place |
| `scan` | `radius` | Scan nearby terrain (stores result) |
| `scan_and_report` | — | Scan + generate text report |
| `look_at` | `q`, `r` | Face toward a hex cell |
| `loop` | `from`, `count` | Jump back to step `from`, repeat `count` times |
| `set_anchor` | `q`, `r`, `layer` (optional) | Set anchor point for relative actions. Defaults to agent position. |
| `move_rel` | `dq`, `dr` | Walk to anchor + offset |
| `place_rel` | `dq`, `dr`, `dlayer`, `block` | Place block at anchor + offset |
| `break_rel` | `dq`, `dr`, `dlayer` | Break block at anchor + offset |

### Block Types

`stone`, `dirt`, `grass`, `sand`, `ice`, `torch`

### How Scripts Execute

1. The script runner processes steps one at a time
2. `move_to` triggers A* pathfinding — the agent walks the path before advancing
3. `place`/`break` happen instantly, then advance
4. `wait` pauses the runner for N seconds
5. `loop` jumps back to an earlier step, decrementing a counter each time
6. When all steps are done, the agent returns to idle

### Journal vs Orders

These are two different things:

**`journal.jsonl`** — event log. Records what happened and when. For review and learning.
```json
{"ts":"2026-03-22T14:30:15","event":"Received order: Build 5-block wall (10 steps)"}
```

**`orders/*.json`** — executable scripts. The actual instructions the agent follows.
```json
{"name":"wall_NE_5","description":"Build 5-block wall","steps":[...]}
```

The journal logs events. Orders drive behavior. Scripts in `orders/` accumulate over time as a library of patterns the agent has executed before.

### Writing Scripts by Hand

1. Create a JSON file in `cache/scripts/my_script.json`
2. Use the format above with `name`, `description`, and `steps`
3. In-game, type `/script my_script` to load and run it
4. The agent will execute each step in sequence

### LLM Script Generation

When you ask the agent to build something complex:
1. The game detects "build"/"construct"/"create"/etc. in your message
2. Sets `script_mode` — tells the LLM to output a `[SCRIPT]...[/SCRIPT]` block
3. The LLM receives the agent's current position (q, r, ground_layer) and surroundings
4. It outputs a JSON script with place/break/move steps
5. The game parses it, saves to `orders/`, and executes immediately

## Agent Files

All agent data lives in `cache/agents/<name>/`:

| File | Format | Purpose |
|------|--------|---------|
| `character.json` | JSON | Appearance, personality, directive |
| `memory.md` | Markdown | Persistent knowledge (editable!) |
| `journal.jsonl` | JSONL | Timestamped events and outcomes |
| `chat_log.jsonl` | JSONL | Raw conversation history |
| `orders/` | JSON files | Generated build scripts |
